# 7★ STAR — 개발 계획 & 모듈별 바이브코딩 가이드 v1.0

> 동반 문서: `PRD.md`, `game_data_dictionary.md`
>
> 이 문서는 **"어떻게 만들 것인가"**의 실전 매뉴얼입니다. 비코더가 바이브코딩으로 진행할 때 **모듈마다 무엇을 준비하고, AI에게 어디까지 요청하고, 어느 완성도를 받아야 하는지**를 전부 정의합니다.

---

## 1. 기술 스택 & 프로젝트 구조

**스택(확정):**
- **빌드:** Vite
- **UI/메뉴/전략맵 HUD:** React
- **전술 전투 캔버스 + 컷인 연출:** Phaser 3 (2D 게임 엔진, 웹용)
- **상태관리:** Zustand (가볍고 AI가 잘 다룸)
- **데이터:** `/src/data/*.json` (데이터 주도 — 코드와 콘텐츠 분리)
- **저장:** localStorage (실제 브라우저에서 동작하므로 사용 가능. *주의: 이 제약은 일부 샌드박스 미리보기에만 해당하고, 실제 배포 게임에선 정상 동작*)
- **배포:** 웹 → Electron(데스크톱) / Capacitor(모바일)

**폴더 구조(권장):**
```
src/
  data/            # 모든 게임 데이터 JSON (데이터 사전 → 여기로)
    ships.json  aces.json  skills.json  resources.json
    items.json  systems.json  enemies.json  research.json  events.json
  game/            # Phaser 전술 전투
    scenes/  systems/  entities/
  ui/              # React 화면
    screens/  components/  hud/
  state/           # Zustand 스토어
  assets/ (→ public/assets)  # PNG·사운드
  core/            # 공용 로직(상성 계산, 데미지 공식 등 순수함수)
```

**핵심 원칙:**
1. **데이터 주도:** 함선·스킬·아이템 수치는 전부 JSON에. 코드에 하드코딩 금지 → 밸런싱·확장이 데이터 수정만으로 됨.
2. **모듈 격리:** 각 모듈은 독립적으로 테스트 가능하게. 한 번에 "전체 게임"을 만들지 않는다.
3. **순수함수 우선:** 상성·명중·데미지 계산은 `core/`의 순수함수로 → AI가 정확히 만들고 테스트 쉬움.

## 2. 바이브코딩 황금률 (먼저 읽기)

1. **한 번에 한 모듈.** "게임 만들어줘"(X) → "MOD-1 전술 그리드만 만들어줘"(O).
2. **데이터를 먼저 준다.** 코드 요청 전에 해당 JSON(데이터 사전에서 발췌)을 AI에게 제공.
3. **완료 정의(DoD)를 함께 준다.** "이게 되면 끝"을 명시하면 AI가 과·미달 없이 만든다.
4. **작게 받고 바로 테스트.** 큰 덩어리로 받지 말고, 동작 확인 후 다음 요청.
5. **컨텍스트 유지.** 프로젝트가 커지면 "현재 폴더 구조와 MOD-X까지 구현됨"을 주기적으로 AI에 요약 제공.
6. **AI 코드는 100% 신뢰 금지.** 직접 실행→에러 복붙→"이 에러 고쳐줘". 이게 정상 워크플로.
7. **리팩터 주기.** 모듈 3~4개마다 "구조 정리·중복 제거"를 한 번 요청.

**완성도 등급(받을 기준):**
- 🟢 **프로토(proto):** 임시값·플레이스홀더 OK, 동작만 확인. (초기 모듈)
- 🟡 **기능완성(functional):** 데이터 연동·엣지케이스 처리, 버그 없음. (대부분 모듈의 목표)
- 🔵 **폴리시(polished):** 연출·UX·밸런스 다듬기. (M5~M6, 출시 전)

### MOD-0 · 프로젝트 셋업 & 데이터 로더 & assetMap
- **목적:** 빈 프로젝트 + JSON 로딩 + 에셋 매핑 골격.
- **준비물:** 폴더 구조(1장), 데이터 사전의 `ships.json` 샘플 1개.
- **요청 예시:**
  > "Vite + React + Phaser 3 + Zustand 프로젝트를 셋업해줘. 폴더 구조는 [붙여넣기]. `/src/data`의 JSON을 앱 시작 시 로드해서 Zustand 스토어에 넣는 `dataLoader`를 만들고, `/public/assets`의 PNG를 키로 참조하는 `assetMap.js`(없으면 이모지 폴백)를 만들어줘. 로딩 화면에 진행률 표시."
- **DoD:** `npm run dev`로 빈 화면+로딩 진행률, 콘솔에 JSON 데이터가 스토어에 적재됨 확인.
- **테스트:** 브라우저 콘솔에서 스토어에 ships 데이터가 있는지 확인.
- **완성도:** 🟢→🟡

### MOD-1 · 전술 전투 그리드 & 이동
- **목적:** Phaser 씬에 그리드(헥스 또는 사각) + 유닛 배치 + 클릭 이동 + 이동범위 하이라이트.
- **준비물:** `ships.json`(MOV·HP 등), 임시 유닛 스프라이트(이모지/사각형 가능).
- **요청 예시:**
  > "Phaser 3로 `BattleScene`을 만들어줘. 12x10 사각 그리드, 데이터의 유닛을 배치하고, 유닛 클릭 시 MOV 값만큼 이동 가능한 칸을 파란색으로 하이라이트, 클릭하면 이동(애니메이션). 지형 데이터(빈칸/소행성/잔해)도 타일로 표시. 좌표·이동은 `core/grid.js` 순수함수로 분리해줘."
- **DoD:** 유닛 선택→이동범위 표시→이동 동작. 지형 타일 구분 표시.
- **테스트:** 정찰함(MOV6)과 전함(MOV2)의 이동범위가 데이터대로 다른지.
- **완성도:** 🟡
- **추가 노트 (2026-06-09, QoL — 모듈 순서 외):** PC 모니터 기준 전투 화면이 작다는 피드백으로 캔버스를
  960×680 → 1200×850, 그리드 셀 `CELL` 64→80으로 확대. `BattleScreen.jsx`의 `WIDTH`/`HEIGHT`와
  `BattleScene.js`의 `CELL` 상수, 그리고 글리프 폰트 크기(터레인·유닛 22px→28px) 정도만 손대면
  반지름·HP바 등 `CELL`에서 비례 파생되는 값들은 자동으로 함께 커진다. 전략맵·함대편성·정비허브
  화면의 `max-width`도 함께 늘려 더 넓은 모니터에 맞췄다(`StrategyMapScreen.css`/`FleetScreen.css`/
  `MaintenanceHubScreen.css`/`App.css`).

### MOD-2 · 전투 해결 (공격·명중/회피·상성·지형·데미지)
- **목적:** 본작의 "전투의 맛" 핵심. 상성·명중/회피·지형 보정이 들어간 데미지 계산.
- **준비물:** 데이터 사전 3장(함선 스탯·상성표), 4.1(명중/회피·지형·부스터 규칙).
- **요청 예시:**
  > "`core/combat.js`에 순수함수로 전투 계산을 구현해줘. 입력: 공격자/방어자 스탯, 거리, 지형(엄폐 여부), 부스터 여부. 규칙: 명중률 = 공격ACC − 방어EVA + 지형보정(엄폐 시 방어EVA+20, 완전노출 시 −20) (+부스터 시 ACC+15). 명중 시 데미지 = ATK − DEF, 클래스 상성 배율 적용(상성표 [붙여넣기]). 결과로 {명중여부, 데미지, 치명}를 반환. 그리고 BattleScene에서 적 클릭 시 이 함수를 호출해 HP를 깎고 데미지 숫자를 팝업으로 띄워줘."
- **DoD:** 정찰함이 전함을 자주 회피, 전함이 맞히면 정찰함 일격사, 순양함이 원거리에서 안전하게 타격 — 상성이 체감됨.
- **테스트:** 같은 상황 100회 시뮬레이션해 명중률이 의도대로 나오는지(AI에게 테스트 스크립트도 요청).
- **완성도:** 🟡 (이 모듈은 밸런스 핵심이라 수치 튜닝 반복)

### MOD-3 · 턴·AP·TP & 기본 적 AI
- **목적:** 턴 순서, AP(행동력), TP(필살기 게이지) 충전, 적의 기본 행동.
- **준비물:** 데이터(AP·TP/turn), 상성 우선순위 규칙(간단).
- **요청 예시:**
  > "턴제 시스템을 만들어줘. 각 유닛은 턴마다 AP만큼 행동(이동=1, 공격=1 등), 행동 종료 후 TP가 충전됨. 클래스별 턴 속도(스피드 순) 반영. 적 AI는 1차로 '가장 약하거나 상성상 유리한 적에게 이동 후 공격' 휴리스틱. 스페이스바로 턴 종료."
- **DoD:** 플레이어 턴↔적 턴 순환, AP 소진 시 행동 불가, 적이 합리적으로 움직임.
- **테스트:** 적이 정찰함 대신 약한 표적을 노리는지, TP가 턴마다 차는지.
- **완성도:** 🟡 (AI는 프로토 수준이어도 됨, 후에 고도화)
- **추가 노트 (2026-06-09, QoL — 모듈 순서 외):** 테스트 중 매 전투를 일일이 손으로 진행하기 번거로워
  '🤖 자동전투' 토글을 HUD 우상단(컷인 토글 바로 아래)에 추가했다. ON 시 플레이어 페이즈에서도
  적 AI와 동일한 `core/ai.js` 휴리스틱(`pickTarget`/`inAttackRange`/`planApproach`)으로 아군이
  자동으로 이동·공격하며, `runAllyAutoTurn`/`takeAllyAutoTurn`이 기존 `runEnemyUnit`/`takeEnemyTurn`
  큐 순회 패턴을 그대로 미러링한다. ON 상태에서는 `handleUnitClick`/`handleCellClick`을 잠가
  수동 입력과 충돌하지 않게 했다. 정식 기획 요소가 아니라 QA/밸런스 확인용 보조 기능이다.

### MOD-4 · 스킬 & 필살기 & 컷인 연출 ★임팩트 핵심
- **목적:** TP 소모 스킬/필살기 + 풀스크린 컷인(정적 일러+이펙트+화면연출+사운드).
- **준비물:** 데이터 사전 6장(스킬·필살기), 에이스 컷인 일러 PNG(없으면 임시 사각형), 이펙트 스프라이트, 사운드.
- **요청 예시:**
  > "TP가 가득 차면 필살기 버튼 활성화. 발동 시 `CutinManager`가: ①시간정지+배경디밍 ②컷인 일러 슬라이드인 ③이펙트 스프라이트 재생 + screen shake + flash ④데미지 적용·숫자팝 ⑤복귀. 데이터의 skill.effect에 따라 단일/광역/관통 처리. 설정에서 컷인 on/off 토글 가능하게(off면 즉시 결과만)."
- **DoD:** 필살기 한 번에 "와" 나오는 연출. 토글 off 시 빠른 진행.
- **테스트:** 카이의 "광휘 돌격" 직선 관통이 데이터대로 적용, 연출 후 게임 복귀 정상.
- **완성도:** 🔵 (이 게임의 셀링포인트라 폴리시까지)

### MOD-5 · 함대·유닛 데이터 & 성장·전직
- **목적:** 함선/에이스 레벨업·능력치 성장·전직(고유 필살기 해금).

- **요청 예시:**
  > "전투 승리 시 XP 분배 → 레벨업 시 데이터의 성장범위로 스탯 상승. 전직 조건(레벨/아이템) 충족 시 상위 클래스로 전직하고 고유 필살기를 해금하는 시스템. 함대 편성 화면(React)에서 보유 함선·에이스·레벨·장착을 확인·관리."
- **DoD:** 전투→레벨업→전직→새 필살기 사용까지 연결.
- **테스트:** 전투함을 조건까지 키워 전직 후 필살기 사용.
- **완성도:** 🟡

### MOD-6 · 전략맵(성단) & 별계 진입

- **준비물:** 데이터 사전 2장(노드·연결·테마).
- **요청 예시:**

- **테스트:** 정복 순서·잠금 로직, 전투↔맵 전환 안정.

### MOD-7 · 정비 허브(모항): 연구·조합·상점·편성
- **목적:** 모항에서의 성장 소비처. 연구 트리, 모듈 조합, 출격 전 상점, 함대 편성.
- **준비물:** 데이터 사전 7~10장(자원·아이템·연구·상점·조합 레시피).
- **요청 예시(쪼개서):**
  > (7a) "연구 화면: `research.json` 트리를 표시, 자원 소비로 노드 해금(시간/즉시 선택). 해금 시 아이템·기능 언락."
  > (7b) "상점 화면: `items.json`에서 판매목록 표시, 스텔라크레딧으로 구매, 함선에 장착(슬롯 제한)."
  > (7c) "조합 화면: 레시피(재료→결과) 표시, 재료 충족 시 제작."
- **DoD:** 연구→아이템 해금→상점 구매→장착→조합으로 신무기 제작까지 연결.
- **테스트:** 자원 부족 시 비활성, 장착이 전투 스탯에 반영되는지.
- **완성도:** 🟡

### MOD-8 · 자원·채굴 & 경제
- **목적:** 자원 보유·획득·소비 흐름, 기본 채굴.
- **준비물:** 데이터 사전 7장(자원 정의), 별계별 채굴원.
- **요청 예시:**
  > "자원 스토어(스텔라크레딧·티타늄·에너지크리스탈·다크매터) + 획득/소비 함수. 특정 별계의 광산 노드를 확보하면 턴/방문마다 자원 산출. (Could) 광산은 매장량이 있어 고갈되면 산출 중단."
- **DoD:** 전투 보상·채굴로 자원 증가, 상점·연구·조합에서 소비.
- **테스트:** 채굴원 확보 전후 자원 산출 차이.
- **완성도:** 🟡 (고갈은 Could)

---

### MOD-9 · 행성 개발 & 해금
- **목적:** 정복한 별계(행성) 개발 → 행성 고유 함선/기술 해금.
- **준비물:** 데이터 사전 2장(별계별 개발 해금 항목).
- **요청 예시:**
  > "정복한 별계에 자원 투자로 개발 단계 상승, 단계 도달 시 해당 별계 전용 함선/기술 해금(데이터의 unlock). Heroes처럼 '이 행성에서만 만들 수 있는' 것을 조선소 목록에 추가."
- **DoD:** 별계2 개발→채굴장 가동→별계5 개발→다크매터 정제 해금 등 데이터대로.
- **완성도:** 🟡

---

### MOD-10 · 히든 유니크·비밀상점·영입·이벤트
- **목적:** 발견·수집·서사의 깊이.
- **준비물:** 데이터 사전 8장(히든 유니크 위치·1회성), 5장(영입 조건), 11장(이벤트).
- **요청 예시(쪼개서):**
  > (10a) "특정 별계의 특정 칸/조건에서 1회성 유니크 아이템 획득(획득 후 영구 사라짐). 데이터의 hiddenUnique 참조."
  > (10b) "조건 충족 시 비밀상점 노드 노출, 유니크 장비 판매."
  > (10c) "별계6에서 조건(예: 적 에이스 무력화) 충족 시 레이븐 영입."
  > (10d) "이동/진입 시 `events.json` 확률표로 이벤트 발생(전투/자원/특별/정찰/평화). 일부 선택지·연쇄 이벤트."
- **DoD:** 히든 유니크 놓치면 못 얻음, 영입 1건 동작, 이벤트 확률·효과 적용.
- **완성도:** 🟡 (영입·비밀상점은 Should)

### MOD-11 · 보스전 & 승패·엔딩
- **목적:** 성단 보스(심연의 파수꾼) 전투 + MVP 엔딩 + 다음 은하 떡밥.
- **준비물:** 데이터 사전 4장(보스 스탯·패턴), 보스 컷인 일러.
- **요청 예시:**
  > "보스 별계 전투: 보스는 고HP·페이즈(HP 50% 시 패턴 변화)·전용 컷인. 격파 시 엔딩 연출 + '다음 은하' 티저. 패배 시 모항 복귀(페널티)."
- **DoD:** 보스 페이즈 전환·격파·엔딩까지.
- **완성도:** 🔵 (클라이맥스라 연출 신경)

### MOD-12 · 저장/로드 & 설정
- **목적:** 진행 저장, 연출 토글 등 설정.
- **요청 예시:**
  > "전체 게임 상태(함대·자원·정복·연구·인벤토리)를 localStorage에 저장/로드. 슬롯 1~3. 설정: 컷인 on/off, 사운드 볼륨, 전투 속도."
- **DoD:** 저장 후 새로고침→이어하기 정상.
- **완성도:** 🟡

### MOD-13 · UI/UX 셸 & 사운드
- **목적:** 타이틀·메뉴·HUD 통합, BGM/SFX 연동, 전체 시각 통일.
- **요청 예시:**

- **DoD:** 처음부터 끝까지 메뉴로 매끄럽게 이동, 사운드 재생.
- **완성도:** 🔵
